--------------------------------------------Quick Overview of Connections--------------------------------------------
-----------> In QML <------------
In the code for the QML object (ex. a gauge), put the following:

Component.onCompleted: {
	backend.connectionName.connect((value) => {
        	console.log("Received:", value);
        	objectNameTimelineCurrentFrame = value;  // Update gauge
	});
}

You will want to modify connectionName (name of the connection to be created) and objectName (name of the object
with which we are connecting) accordingly. Additionally, the console.log() function is not necessary, but is
nice when debugging.


----------> In python <----------
Do one of these:
class MainWindow(QObject):
    def __init__(self):
    
You might also want to put your qtimers there. But mainly, we initialize connections to qml here. It is done like
this: connectionName = Signal(int)

That is an example, where the connection is name "connectionName" and receives a integer. They will all be different,
depending on how you set them up.

Later, in one of your qTimers, you can send data to qml. Here is an example:

self.connectionName.emit(int(123))

Here we send the integer "123" to connectionName.

When making connections, it is best to give them verbose names. Nobody wants to track down what "updateCLUPEDPOSV"
or "updateTRRS" means. These were old names used last year, based on CANBUS IDs. If you do this I will whack your
pp and you will regret! So avoid it. Names like "updateEngTemp" make much more sense - it updates the engine
temperature. Do this, and you will have a fun, wonderful, and generally enjoyable experience with QML and QT.
